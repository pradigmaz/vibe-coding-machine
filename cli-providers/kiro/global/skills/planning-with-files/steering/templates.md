# Planning with Files: Готовые шаблоны

Копируй эти шаблоны для быстрого старта с planning-with-files.

## Шаблон 1: task_plan.md (Полный)

```markdown
# Task Plan: [Краткое описание задачи]

## Цель
[Одно предложение описывающее конечное состояние]

## Текущая фаза
Phase 1

## Фазы

### Phase 1: Requirements & Discovery
- [ ] Понять намерение пользователя
- [ ] Определить ограничения и требования
- [ ] Задокументировать находки в findings.md
- **Status:** in_progress

### Phase 2: Planning & Structure
- [ ] Определить технический подход
- [ ] Создать структуру проекта если нужно
- [ ] Задокументировать решения с обоснованием
- **Status:** pending

### Phase 3: Implementation
- [ ] Выполнить план шаг за шагом
- [ ] Записать код в файлы перед выполнением
- [ ] Тестировать инкрементально
- **Status:** pending

### Phase 4: Testing & Verification
- [ ] Проверить что все требования выполнены
- [ ] Задокументировать результаты тестов в progress.md
- [ ] Исправить найденные проблемы
- **Status:** pending

### Phase 5: Delivery
- [ ] Проверить все выходные файлы
- [ ] Убедиться что deliverables полные
- [ ] Доставить пользователю
- **Status:** pending

## Ключевые вопросы
1. [Вопрос на который нужно ответить]
2. [Вопрос на который нужно ответить]
3. [Вопрос на который нужно ответить]

## Принятые решения
| Решение | Обоснование |
|---------|-------------|
|         |             |

## Встреченные ошибки
| Ошибка | Попытка | Решение |
|--------|---------|---------|
|        | 1       |         |

## Заметки
- Обновляй статус фазы: pending → in_progress → complete
- Перечитывай план перед важными решениями (манипуляция вниманием)
- Логируй ВСЕ ошибки - они помогают избежать повторения
- Никогда не повторяй неудачные действия - меняй подход
```

## Советы по адаптации шаблонов

### 1. Количество фаз

- **Простые задачи** (1-2 часа): 3 фазы
- **Средние задачи** (2-8 часов): 5 фаз
- **Сложные задачи** (>8 часов): 7+ фаз

### 2. Детализация

- **Начало задачи**: Используй полные шаблоны
- **Знакомая задача**: Используй минимальные шаблоны
- **Критичная задача**: Добавь дополнительные секции

### 3. Кастомизация

Добавляй секции специфичные для задачи:
- **API Integration**: Секция "Endpoints"
- **UI Development**: Секция "Design Decisions"
- **Database**: Секция "Schema Changes"
- **Performance**: Секция "Benchmarks"

## Примеры кастомизации

### Для API Integration

```markdown
## API Endpoints
| Endpoint | Method | Purpose | Status |
|----------|--------|---------|--------|
| /api/users | GET | List users | Pending |
| /api/users/:id | GET | Get user | Pending |

## Authentication
- Type: JWT
- Storage: httpOnly cookies
- Refresh: 7 days
```

### Для UI Development

```markdown
## Design Decisions
| Component | Library | Rationale |
|-----------|---------|-----------|
| Button | Custom | Brand consistency |
| Modal | Radix UI | Accessibility |

## Responsive Breakpoints
- Mobile: < 640px
- Tablet: 640px - 1024px
- Desktop: > 1024px
```

### Для Database Changes

```markdown
## Schema Changes
| Table | Change | Migration |
|-------|--------|-----------|
| users | Add avatar_url | 001_add_avatar.sql |
| posts | Add published_at | 002_add_published.sql |

## Indexes
- users.email (unique)
- posts.published_at (btree)
```

## Заключение

Эти шаблоны - отправная точка. Адаптируй их под свои нужды, но всегда сохраняй:

1. **Цель** (одно предложение)
2. **Фазы** (с чекбоксами и статусами)
3. **Принятые решения** (таблица)
4. **Встреченные ошибки** (таблица)

Это минимум для эффективного planning-with-files workflow.
